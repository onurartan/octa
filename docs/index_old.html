Asset & Avatar Microservice

Stop using complex object storage for simple assets. Octa provides a unified API to generate algorithmic avatars on-the-fly and store static user uploads directly in SQLite Blobs for maximum speed and portability.

Try Playground How It Works

SQLite Blob Storage

Octa uses SQLite in WAL Mode as a high-performance object store. For small files (like avatars/thumbnails), storing blobs in SQLite is often faster than the filesystem and simplifies backups to a single file.

Key-Value Mapping

Decouple your file IDs from public URLs. Map a single uploaded blob to multiple keys (e.g. user-123, admin-profile). Assets are retrieved instantly via these keys, not complex file paths.

Algorithmic Generation

If no custom asset is found for a key, Octa automatically falls back to generating a unique, consistent avatar based on the key string (Initials, Gradients, Shapes).

API Documentation

Reference for the Avatar Generator and Blob Storage Engine.

1

Generation API

Algorithmic creation for users without uploaded photos.

The engine generates deterministic colors and shapes based on the input seed. Use the theme parameter to control the visual output style.

# Basic Endpoint
GET /avatar/{seed}
# With Theme
GET /avatar/onur?theme=gradient/retro
Param Description
theme Format: style/palette.
Styles: gradient, color, soft.
bg Force background color (Hex or Name).
color Force text color.
rounded 0-100 or 'true'.
2

Blob Store API

Static asset storage and retrieval via SQLite.

Store assets in the internal SQLite engine using the /upload endpoint (Requires X-Secret-Key). Ideal for user profile pictures and static site logos.

Upload Asset (cURL)

POST /upload
curl -X POST {{.BaseURL}}/upload \ -H "X-Secret-Key: YOUR_SECRET" \ -F "avatar=@profile.jpg" \ -F "keys=user-123,my-profile"

Retrieve Asset

Access stored blobs using any mapped key. The server automatically handles MIME types and caching.

GET /u/{key}
# Example
GET /u/user-123
GET /u/my-profile
Note: Blob serving serves the raw binary data stored in the database. Use this for avatars uploaded by users.
